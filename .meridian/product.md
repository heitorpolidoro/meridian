# Product Vision: Meridian Project Orchestrator

## 1. Macro Objective
To create a high-integrity, local-first orchestrator that enforces the **Software Development Standard (SDS)** while reducing the cognitive load of managing multiple AI-driven development tracks.

## 2. Strategic Goals
- **Autonomy with Governance:** Enable AI agents to work autonomously within strict SDS-defined guardrails.
- **Context Precision:** Eliminate "context pollution" by isolating agent knowledge to only what is necessary for the current task/track.
- **Human-in-the-Loop Orchestration:** Provide a seamless control center for a human orchestrator (Heitor) to direct, audit, and sign off on agent work.

## 3. Core Principles
- **Local Sovereignty:** All project data, metadata, and agent logs stay on the local machine.
- **Traceability:** Every code change must be traceable back to a specification and a plan.
- **Zero-Waste Context:** Agents only receive context they *must* have to succeed, protecting security and reducing token usage/noise.

## 4. Success Metrics (Overall)
- **SDS Compliance:** 100% of managed tracks pass automated SDS audits.
- **Efficiency:** 50% reduction in time spent on manual "context prepping" for agent sessions.
- **Reliability:** Zero "out-of-scope" changes committed by agents.
